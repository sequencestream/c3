# 0022 — 规范消息不扩展（沙箱/Checkpoint 通过事件总线）

- **Status:** accepted
- **Date:** 2026-06-09

## Context

沙箱集成（sandbox 容器内运行 agent）引入了两个新的跨层关注点：

1. **容器生命周期** — 启动/停止容器的信号需要从 run 生命周期到达内核沙箱层。
2. **Pre-approve checkpoint** — 在自动批准工具调用前，需要检查沙箱工作区的文件变更。

这两个关注点都需要跨层通信。一个直观的方案是扩展规范消息（canonical message）协议，让 sandbox 相关的状态（容器 ID、checkpoint 结果等）作为新字段或新 block 类型承载在规范消息流中。

但扩展规范消息有代价：它增加了 wire 协议的复杂度，将服务器内部的状态管理与前端数据结构耦合，并为后续无法预见的上层功能打开了"再扩一个字段"的先例。

## 选项

### 1. 扩展规范消息

给规范消息新增可选字段（如容器 ID、checkpoint 结果），让沙箱状态随消息流传递到前端。

_Pro:_ 前端可以从消息里直接知道容器 ID，无需额外查询。
_Con:_ 污染了规范信封的通用性——规范消息的核心职责是承载 vendor 无关的 agent 会话内容（文本、工具调用、结果），不是服务器内部基础设施状态。
_Con:_ 前端不需要知道容器 ID、checkpoint 结果——这些是服务器内部决策信号。
_Con:_ 开了一个先例：后续任何跨层关注点（资源监控、网络状态、缓存命中率）都会要求扩规范消息，导致协议膨胀。

### 2. 使用进程内事件总线 (selected)

沙箱状态通过已在运行的进程内事件总线（ADR-0018）在内核层传递，不扩展规范消息。

- **容器生命周期**：run 启动器在 vendor fork 前启动 sandbox，由会话运行时持有容器句柄与停止闭包的引用。sandbox 的生命周期绑定到会话运行时的生命周期，而不是消息流。run 结束/运行时移除时通过闭包回调停止容器。
- **Checkpoint**：审批桥提供一个可选的预批准 checkpoint 方法，直接通过沙箱驱动的复制能力操作容器，结果通过内存返回，不进消息流。
- **错误通知**：如果 sandbox 启动失败，记入日志（非致命，run 继续执行），或者未来通过事件总线发布一个 sandbox 错误事件。

_Pro:_ 规范消息不扩展——它保持纯粹的会话内容承载职责。
_Pro:_ 前端完全无感知（前端不需要知道 sandbox 存在）。
_Pro:_ 与现有架构完全一致——会话运行时已经是进程级 registry，事件总线已是跨层通信通道。
_Con:_ 前端无法在 UI 中显示 sandbox 状态——这是一个 feature，不是 bug。sandbox 是一个透明的基础设施层。

### 3. 新增专用 wire frame

新增一个服务端到客户端的 wire 消息变体（如携带容器 ID 的 sandbox 状态帧），通过现有 WebSocket 通道下发 sandbox 状态。

_Pro:_ 前端可以获得 sandbox 状态（如果将来需要显示）。
_Con:_ 前端目前不需要；这会增加协议复杂度。
_Con:_ 为未来可能不需要的功能增加 wire 协议。
_Con:_ 与 ADR-0006（run 与连接解耦）的精神不一致——sandbox 是服务器端资源，不是连接级状态。

## 决策

**不扩展规范消息**。新引入的沙箱关注点通过以下已有机制承载：

| 关注点                 | 机制                                                             |
| ---------------------- | ---------------------------------------------------------------- |
| 容器生命周期           | 会话运行时持有容器句柄 + 停止闭包                                |
| 容器清理               | run 结束 / 运行时移除时调用停止闭包                              |
| Pre-approve checkpoint | 审批桥的可选预批准 checkpoint 方法，返回内存中的 checkpoint 结果 |
| Checkpoint 文件检查    | 沙箱驱动的容器文件复制能力（`docker cp`）+ 内存文件列表          |
| Docker 不可用          | 记入日志后降级（run 继续执行），未来可选事件总线事件             |
| 前端感知               | 无——sandbox 是透明基础设施                                       |

## 后果

- 规范消息不承担 sandbox 状态职责。
- 前端无 sandbox 相关变更。
- 后续 Phase 2（断点续传、checkpoint-restore）需要序列化容器状态到磁盘时，将引入新的持久化机制（而非扩展 wire 协议）。
- 如果将来需要在前端显示 sandbox 状态，应新增专用服务端到客户端 wire frame 而非扩展规范消息。
