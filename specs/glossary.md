# Glossary

Business and technical terms used across c3 specs. Defined once here; domain docs
reference these rather than redefining them.

- **c3**: Claude Code Center. The application: a local web UI that gates Claude Code's tool use through a browser.
- **Agent run**: A single `query()` invocation driven by one user prompt. Streams assistant text, tool activity, and permission requests until it completes or errors.
- **Permission request**: A server‚Üíclient event asking the user to authorize a specific tool call (tool name + input). Emitted from the SDK `canUseTool` callback. Identified by a `requestId`.
- **Permission decision**: The user's answer to a permission request: `allow` or `deny`.
- **Permission mode**: The policy the SDK uses to decide which tools need a request. One of `default`, `auto`, `plan`, `acceptEdits`, `bypassPermissions`. Persists for the life of a connection.
- **canUseTool**: The SDK callback c3 supplies. Invoked for sensitive tool calls; c3 routes the question to the browser and returns the resolved decision.
- **Sensitive tool**: A tool the SDK classifies as needing approval under the active mode (e.g. `Write`, `Edit`, dangerous `Bash`). Read-only tools are auto-allowed by the SDK and never produce a request.
- **Auto-deny**: The outcome when a pending permission request is cleared because the run was stopped (`stop_run` / delete / workspace removal) before the user decided. There is no timeout-based auto-deny ‚Äî an unanswered request blocks indefinitely, and switching the view never denies it.
- **Wire protocol**: The JSON message contract over the `/ws` WebSocket. Two discriminated unions: `ClientToServer` and `ServerToClient`, defined in `shared/src/protocol.ts`.
- **Project directory**: The seed workspace directory passed as `--project` (defaults to the current working directory), used as the SDK `cwd`. Claude reads and writes files relative to it.
- **Session**: A Claude conversation in a workspace. Its execution is owned by a process-wide **Session Runtime**; a connection only **views** one session at a time. Runs are not bound to a connection (ADR 0006).
- **Session Runtime**: The process-wide owner of one session's run (`server/src/runs.ts`): its abort/handle, a `baseline + buffer` of wire events for replay, the current viewers, and live status. Shared across connections; survives switching/disconnect.
- **Session status**: A session's live run state: `idle`, `running`, `awaiting_permission`, or `team`. Broadcast to all connections via `session_status` to drive sidebar badges.
- **Viewed session**: The single session a connection currently watches. The next `user_prompt` from that connection runs against it; switching the view never stops the previous session's run.
- **Run handle**: Live controls for an in-flight run handed to the WS handler (`setPermissionMode`), letting a mode change apply mid-run.
- **Interrupt / abort**: Stopping an in-flight run via the SDK `query.interrupt()`. Triggered only by `stop_run`, `delete_session`, or `remove_workspace` ‚Äî never by switching the viewed session or closing the connection.
- **Static embed**: The web bundle inlined into the compiled binary (`server/src/static-embed.ts`, generated and gitignored). Served when no filesystem `web/dist` is present.
- **claude CLI**: The host-installed `claude` executable the SDK spawns to run the agent. Resolved from `$CLAUDE_PATH` or the PATH (via the ProcessLauncher). A hard runtime dependency ‚Äî the Claude instance of the per-vendor **host binary** intent (ADR-0012).
- **turn_end**: The terminal server‚Üíclient event for one prompt‚Üíresult turn, with `reason` `complete` (incl. a stopped run) or `error`. Never means the session ended ‚Äî it stays alive for the next prompt.

The unit the intent list, refine, and launch-development all operate on (ADR 0007). Renamed from **Requirement** (requirements‚Üíintents, PR-2). **Disambiguation:** the domain **Intent** (capitalized; a persisted work-unit entity) is NOT the lowercase word _intent_ used elsewhere for unrelated concepts ‚Äî a Vue child component's event _intent_ (a user-action emit), the ADR-0011 permission _intent_, the consensus vendor-free _intent_, and the orchestrator's next-subtopic / broadcast _intent_. Those are same-word / different-context (cross-referenced here) and are deliberately untouched by this rename.

The run never auto-completes the intent.
- **Ëá™Âä®Êèê‰∫§ lint Ëá™ÊÑà (Auto-commit lint self-heal)**: When the automation orchestrator's `done` auto-commit is rejected by a **pre-commit lint hook** (`commitAndPush` returns `failure: 'commit-hook'`), it self-heals instead of hard-stopping by handing the failure to the **dev agent once** ‚Äî lint toolchains differ per project, so there is no portable fix command. - **c3.db**: The SQLite intent ledger at `~/.c3/c3.db`. Distinct from the registry's `state.json` (under `~/.claude/c3/`); accessed through a cross-runtime driver adapter (`node:sqlite` / `bun:sqlite`) that fails soft so c3 still boots without it (ADR 0007).
- **Streaming input**: The SDK mode where `query()`'s prompt is a controlled async-iterable (`InputStream`) rather than a one-shot string. Pushing a turn appends to the same live session; closing the stream ends the query. c3 runs **all** sessions this way (ADR 0008): it keeps the lead process alive for agent teams, makes `setPermissionMode`/`interrupt` effective, and reproduces one-shot behaviour for normal sessions by closing the stream on `result`.
- **Agent team**: A Claude Code SDK arrangement where a **team lead** delegates work to **teammates**. c3 detects it at runtime when a team tool (`TeamCreate`, `SendMessage`, or a background `Agent`) is used, upgrading the session to a **persistent session** (AS-R14, ADR 0008). **Claude-locked** (AS-R21, 2026-06-06-006): a session becomes a team only when its vendor has `streamingPush` (a resident lead running in-process `TeamCreate`/`SendMessage`) ‚Äî only Claude does; a non-Claude session is never marked `team`. A heterogeneous teammate is a deferred capability.
- **Team lead**: The primary agent of an agent team ‚Äî the `query()` run c3 owns. In a team session its process stays alive between turns (it does not exit on `result`) to coordinate teammates; c3 keeps the run in flight until the user explicitly stops it.
- **Teammate**: A delegated agent spawned by the team lead, e.g. a background `Agent` (`run_in_background: true`) that runs detached and reports its result back to the lead asynchronously. Killing the lead process would orphan teammates ‚Äî the reason the lead must stay alive (ADR 0008).
- **Persistent session**: A session whose run stays in flight between turns rather than ending each turn. Currently only a `team` session: the lead process is alive between turns, status holds at `team` (not `idle`), and it ends only on explicit user stop (`stop_run` / `delete_session` / `remove_workspace`).
- **team_upgraded**: The one-shot server‚Üíclient event (and `SessionStatus` `team`) marking a session as a persistent agent team. Recorded in the session buffer so reconnect replay shows it; drives the team badge and the always-enabled composer in the UI.
- **Vendor adapter**: One agent vendor's implementation of the neutral abstraction (`adapters/<vendor>/`): its `AgentDriver`, `ApprovalBridge`, `SessionStore`, and `AdapterCapabilities`. c3 selects one by `VendorId` (`claude`/`codex`/`opencode`) and drives it through the neutral faces only. Three adapters exist: Claude (reference, all six capabilities true), OpenCode (full integration, `perToolApproval` true), and Codex (read-only advisor seat, ledger all-false per 008 NO-GO) (ADR-0011).
- **AgentDriver**: The neutral lifecycle + streaming face of a vendor: `start(opts)` returns an `AgentRun` exposing `sessionId()`, `messages()` (a canonical stream), and `abort()` (required), plus capability-gated optional controls. Generalizes the Claude `query()` loop (ADR-0011).
- **ApprovalBridge**: The neutral approval channel ‚Äî intercept ‚Üí suspend ‚Üí write back ‚Äî via `onRequest(handler) ‚Üí disposer`. For vendors with `perToolApproval` the handler fires per tool (Claude resolves the blocking `canUseTool`; OpenCode POSTs a REST decision); Codex has no live point and degrades to launch-time policy (ADR-0011).
- **SessionStore**: The neutral session-history face that hides the dirtiest coupling (Claude's direct JSONL reads, OpenCode REST, Codex thread items) behind `list`/`read`, returning canonical `CanonicalMessage[]`; optional `rename`/`delete` (ADR-0011).
- **AdapterCapabilities**: The probed ledger of a vendor's optional/degradable abilities ‚Äî two parts (ADR-0011, amended 2026-06-07): **(a)** exactly six **boolean** live-run flags: `interrupt`, `setActionMode`, `streamingPush`, `inProcessMcp`, `forkSession`, `perToolApproval`; **(b)** a structured sub-ledger **`sessions: SessionCapabilities`** for the session-lifecycle operations (`list` / `read` / `resume` / `rename` / `delete`), each a 4-state `CapabilityState` (`'none'` / `'partial'` / `'full'` / `'temporarily-unavailable'`) ‚Äî a boolean could not tell `none` (structural NO) apart from `temporarily-unavailable` (mechanism exists, not currently reachable), and that distinction is what the UI renders. Required capabilities carry no flag. The upper layer probes a flag (and method presence) before reaching for a divergent control, degrading when absent; the session sub-ledger's states drive row-action gating by state, with **zero `if (vendor === ‚Ä¶)`** (ADR-0011).
- **CapabilityState**: One of the four values a session-lifecycle operation is graded by (ADR-0011 amendment 2026-06-07): `'none'` (structurally absent ‚Äî the vendor's SDK has no such API, e.g. Codex's `list`/`read`), `'partial'` (supported with loss ‚Äî lossy resume-and-replay reconstruction), `'full'` (first-class support, the Claude reference grade), `'temporarily-unavailable'` (mechanism exists but is not currently reachable ‚Äî OpenCode's REST write-back for `rename`/`delete` is the same shape as a briefly-down remote server). Lives on `SessionCapabilities`; the wire carries the per-vendor matrix on the top-level `settings.sessionCapabilities: Record<VendorId, SessionCapabilities>`.
- **perToolApproval**: The capability flag for in-the-loop, per-tool runtime approval. `true` for Claude and OpenCode; **`false` for Codex** (008: stdin closes after dispatch, no write-back half-channel) ‚Äî the addition beyond Claude's five proprietary controls (ADR-0011).
- **Canonical message**: The vendor-spanning message envelope (ADR-0011, per the 010 diff): `vendor` tag + `sessionId` (unconditional) + discounted `role`/`blocks`/`ts`/`turnId?` + two-level `vendorExtra` overflow. Blocks are append-with-upsert; a tool's return is **embedded** on its `tool_use` block (`result?`), not a standalone block. **Promoted to the wire** (`shared/protocol.ts`, SDK-free) in ADR-0013 ‚Äî the kernel re-exports it; the wire gains only a `vendor` dimension, never a per-vendor schema.
- **ActionMode √ó ToolGate (neutral permission grid)**: The 2-axis replacement for Claude's five-way `PermissionMode` (ADR-0011): `ActionMode{plan,build}` (what the run may do) √ó `ToolGate{always-ask, on-sensitive, trusted-prefix, never-ask}` (how tools are gated). Each adapter translates its native mode(s) in; the grid does not round-trip 1:1 (Claude `auto`'s bias and `always-ask` have no exact peer). The neutral policy is `(toolName, input, ctx) ‚Üí allow|ask|deny`.
- **Host binary (host CLI)**: The vendor CLI an agent type runs as a subprocess (`claude`/`codex`/`opencode`). **It can NOT be packed into c3's single binary** ‚Äî the binary ships c3 only ‚Äî so each agent type requires its host CLI installed on the host PATH. A missing one means that agent type is unavailable: a product convention, not a bug, surfaced as install guidance (ADR-0012).
- **ProcessLauncher**: The vendor-agnostic host-binary probe (`server/src/kernel/agent/process/launcher.ts`): `resolve(vendor) ‚Üí absolute path | null` (precedence `$<PATH_ENV>` ‚Üí PATH probe ‚Üí cache), a `HOST_BINARIES` table (binary name / `*_PATH` override / install hint per vendor), and `probe`/`probeAll` for the boot health check. The **first capability gate** ‚Äî a `null` short-circuits before any adapter or capability (ADR-0012).
- **Adapter registry**: `server/src/kernel/agent/adapters/registry.ts`: maps each implemented vendor to its factory (`VENDOR_FACTORIES`, `{claude}` today) and turns the ProcessLauncher probe into `resolveAvailableAdapters() ‚Üí { available, missing }`. Probes **before** constructing each adapter; an unresolved binary lands in `missing` (with install hint) and is never built. `available` is the kernel's source of truth for available agent types; `logHostBinaryHealth()` logs it at boot (ADR-0012).
- **AdapterCapability (enum)**: The wire-facing capability enum (`shared/protocol.ts`, ADR-0013) naming the six optional/degradable capabilities: `interrupt`/`setActionMode`/`streamingPush`/`inProcessMcp`/`forkSession`/`perToolApproval`. The kernel's `AdapterCapabilities` boolean ledger is keyed by exactly these names; a compile-time assertion pins the two so they cannot drift.
- **Block upsert (two-form)**: The rule that lets both vendor message forms share one model (ADR-0013): blocks are keyed by `(sessionId, block.id)` and **upserted**, not append-only. Claude's whole-message frame and Codex's incremental `ItemUpdated` both collapse to "revise in place, don't stack"; a tool's return back-fills its `tool_use` monotonically. Implemented by `CanonicalAccumulator` / `upsertBlock` (`adapters/canonical-accumulator.ts`).
- **c3SessionId**: The opaque, vendor-free session handle (`"c3s_" + sha256(vendor \0 vendorSessionId)[:32]`, ADR-0013) ‚Äî the only session id that crosses out of the kernel. Deterministic (no persistence needed) and containing neither the vendor name nor the raw id as a substring, so a vendor id never leaks into a URL or storage key. Resolves to a `SessionRef { vendor, vendorSessionId }` kept inside the kernel.
- **SessionAccessor**: The **read-only** lazy-normalizing union over the available vendors' `SessionStore`s (`kernel/agent/session/accessor.ts`, ADR-0013): `list` merges across vendors (native id hidden in `vendorExtra`), `read` routes to the owning store via a lazily-built `c3 ‚Üí ref` index. Native stores stay the source of truth ‚Äî no double-write; the index is a rebuildable runtime cache (Â≠òÂÇ®ÂΩ¢ÊÄÅÂΩí‰∏Ä„ÄÅ‰ΩçÁΩÆ‰∏çÂΩí‰∏Ä).
- **Session‚Üíagent binding (two-key)**: The per-session record of which agent a session runs on, split into two key spaces (ADR-0015, agent-config AC-R16): a mutable **pending intent** and a settled **session fact**. Persisted at `~/.c3/state.json` v2 (`pendingIntents` + `sessionAgents`); `getSessionAgentId` reads both. Distinct from `c3SessionId` / `SessionAccessor`, which are about reading transcripts.
- **Pending intent**: The mutable half of the binding (ADR-0015): `pendingId ‚Üí { agentId, createdAt }` ‚Äî which agent a not-yet-run session wants. Editable/clearable; copied to a fact at first bind and dropped, else reaped by the janitor (`cleanupStalePendingIntents`) after 7 days. Clearing/reaping an intent never touches the fact space, so it can't orphan a fact.
- **Session fact / vendor ownership**: The settled half of the binding (ADR-0015): `realId ‚Üí { agentId, vendor }` ‚Äî the agent a real session ran on plus its **frozen vendor**. The vendor is the immutable invariant: a session's transcript lives only in that vendor's native store (c3 never stores session content), so the vendor cannot change. Same-vendor agent swaps are allowed (`changeSessionAgentFact` ‚Üí `true`); cross-vendor changes are rejected (`false`). Cross-vendor Fork / **replay-seed** stays deferred (ADR-0011).
- **release (build)**: The thin orchestration layer over the existing build/binary primitives that produces multi-platform artifacts (`pnpm release:build`). It does not replace `pnpm build` or `pnpm binary`; it sequences three race-free phases ‚Äî Phase0 web build (once) ‚Üí Phase1 generate-static-embed (once) ‚Üí Phase2 parallel `bun --compile` fan-out (read-only). See `specs/non-functional/release.md`, ADR-0010.
- **channel**: A named distribution track for release artifacts (e.g. `stable`, `nightly`). Determines artifact naming/suffix and which version is published. **Placeholder** in release 1/7 ‚Äî the orchestrator builds without a channel concept yet.
- **hardened (tier)**: A build-hardening level passed to `buildTarget({ harden })` ‚Äî the ladder from `default` up through stripped symbols, reproducible builds, signing, and notarization. **Placeholder** in release 1/7: only the `default` tier is honored; other values warn and fall back.
- **version SoT**: The single source of truth for the artifact version and its build-time injection (currently the package manifest; `--version` ‚Üí `0.1.0`). A unified version-SoT + git-tag/channel injection is a **placeholder** for a later release wave.
